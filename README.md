etail organizations generate large volumes of data through point-of-sale and e-commerce 
systems. That data becomes valuable only when it is validated, transformed, and analyzed with a 
clear business objective. In a typical retail analytics workflow, a data analyst defines KPIs, 
prepares the dataset, validates totals using SQL, and builds dashboards that help managers take 
day-to-day decisions. This project mirrors that practical workflow using RoyalMart’s 
transaction-level dataset so that every metric and insight can be traced back to records and 
verified. 
A unified KPI framework is essential for decision support. Without consistent definitions, 
different teams can compute the same metric in different ways and reach conflicting conclusions. 
To avoid that, a KPI catalog should define each metric, its formula, refresh frequency, and 
source fields. In this report, KPIs such as Net Sales, Profit, Profit Margin, Average Order Value 
(AOV), Discount Rate, Online Share, CSAT, and Return Rate are defined and used consistently 
across SQL validation and Power BI visuals.
